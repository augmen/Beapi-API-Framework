Controller Implementation

Without Boomstick™, you may find yourself having to update your controllers, domains, security, urlmappings, etc just to install API's. This is NOT THE CASE with BOOMSTICK™. Boomstick™ makes API development simple, easy, fast and scalable. All you have to do is plug and play and Boomstick™ takes care of the 'REST'.

The only requirement is that in your controller when returning a 'resource', you have to return your data under the name of controller. So if your Controller is named 'FredController' and you are returning data from a 'show' method, you would return it like so:

return ['fred':data]
