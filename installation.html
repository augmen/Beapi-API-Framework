Requirements

The BoomStick™ API Framework has the following project requirements:

  • Grails 3.1.1 (or higher)
  • spring-security-core 3.0.3 (or higher)

NOTE: You may notice in the sample project that several rest plugins and web/mvc plugins have been left out from the dependencies. This is deliberate; due to out API abstraction layer, they can cause confusion and unnecessary slowness in the application.

Configuration

In your build.gradle file, you will need the following plugins:

apply plugin: "spring-boot" apply plugin: "org.grails.grails-web"

You will also need the following dependencies:

dependencies { compile("org.grails.plugins:api-framework:0.1-SNAPSHOT") compile "org.springframework.boot:spring-boot-starter-logging" compile "org.springframework.boot:spring-boot-autoconfigure" compile "org.grails:grails-dependencies" // INSTALL DATABASE DRIVER OF YOUR CHOICE; HERE ARE EXAMPLE //runtime 'mysql:mysql-connector-java:5.1.29' //compile 'org.grails.plugins:mongodb:5.0.2' //compile "org.mongodb:mongodb-driver:3.0.2" //runtime 'org.springframework.data:spring-data-mongodb:1.8.1.RELEASE' provided 'org.springframework.boot:spring-boot-starter-tomcat' compile 'org.grails.plugins:spring-security-core:3.0.3' compile "org.grails.plugins:hibernate" compile "org.hibernate:hibernate-ehcache" testCompile "org.grails.plugins:geb" testCompile "org.grails:grails-plugin-testing" }

Sample Project

All communication logic and data are abstracted away from the controllers and services leaving the controllers to only deal with retrieving and returning a resource. This makes API development INCREDIBLY simple. You simply have to:

  • Install the plugin
  • Build a JSON IO State file for each controller

There are NO ANNOTATIONS to be set, NO SERVICES to rely upon, NO CHECKS to be made. The IO State file acts as a properties file for endpoint rules for ALL services in your architectures and is handled at the communication layer NOT at the business logic. This makes it faster and more scalable and reduces code needed to create API's.
